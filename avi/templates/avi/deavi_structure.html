DEAVI Structure

Home Algorithms Queries Data Help

1 - Home

The home page is the most important element of DEAVI since it is the starting point of the user’s activity. It consists of a header, a side menu, a main container and a footer. Because this structure is maintained in the other pages of the application, we will explain these components in detail in this section.



1.1 - Header

It has three interactive elements. The first one is located to the left where the acronym of the application, DEAVI (Dynamic Evolution Added Value Interface) and the current version are shown. When the user clicks on this item, the home page is loaded. The other element is located on the right. It is the logo of the EU commission which is linked to the EU official website.



1.2 - Side Menu

The side menu is an element used in graphical interfaces to access the information contained in the application in an orderly manner. The DEAVI side menu has four components: Algorithms, Queries, Data and Help. In the following sections, each of them will be explained in detail.



When the user clicks on one of the elements, a new drop-down menu with the available options is displayed



1.3 - Footer

The footer is the area that occupies the bottom of the web application.



It is divided into three parts:

  • Partners: In this section, the logos of the members of the projects and the EU Commission are shown. Each logo is linked to the official site of each member.



  • Pages: The central section is a sitemap, that is, displays the structure of the links contained in the website that define the navigation within the application. This allows the user to visualize and access all the content of the site in a direct way.



  • Our offices: The right section collects the contact information about Quasar Science Resources S.L., Spanish SME that has developed DEAVI. DEAVI users can contact QSR to report problems or issues they may have with the application.



1.4 - Main Container

The last section in DEAVI is the main container. This component is common to all sections, so the information shown depends on the section visited using the side menu. In the case of the home page, a brief description of DEAVI and the SFM project within which the application has been developed are shown. In addition, the user can visit the project website, https://starformmapper.org/ , by clicking on the central image.



2 - Algorithms

The “Algorithms” section in the side menu collects the set of algorithms developed by the scientists and the QSR team as well as the status of their execution. This section has two subsections: “Start” and “Pipeline”

2.1 - Start

In this subsection the user can select the algorithm to execute. DEAVI currently contains simple algorithms developed to test the application. When the user clicks on an algorithm, its input and information fields are displayed. For example,



For more information on the available algorithms and how to use them, see Section Tutorials.

2.2 - Pipeline Status

When an algorithm has been launched, the user can monitor the status of the execution on this page as well as the obtained results



At the very top, the first element of this section consists of a search engine where a certain execution can be located,



The second element displayed is a table where the available executions are shown. The columns are:

  • Name: Name of the execution.

  • Date: Date of the execution.

  • Status: Status of the execution (STARTED, SUCCESS or FAILURE).

  • Info: this parameter gives information on whether the algorithm has errors in the syntax (Script has issues) or not (ok).

  • Relaunch: It allows to launch the execution again with the same configuration, where some parameters can be changed.

  • Abort: Delete the execution.



Note that the user can sort the executions by Name, Date and Status by clicking on each label in the table. This feature is maintained in any of the tables of the same style present in the application. When the execution of the algorithm has been correct, the user can visualize the parameters used as well as the results of running the algorithm by clicking on the Name of the algorithm.



The last element of this section is the pagination where, depending on the number of executions, the user can change page by clicking on the different buttons.

3 - Queries

DEAVI allows to make queries to the archives of the space missions GAIA and Herschel. Although these queries can be made through the official ESA web interfaces, in this case, they will be faster because the data is not downloaded to the user’s computer, that is, there will be no movement of data through the network.

3.1 - Gaia

The user can make queries to Gaia by clicking on “Gaia”. The application has four different ways of making queries:

  • Name

The application allows you to make queries to the Gaia archive by name using the SIMBAD/NED identifier . For example, if we enter M45, the query to star cluster Messier 45 will be made.



Query by Name: At the bottom of the form, the user must choose the shape of the query (cone, box or polygon), the version of the Gaia data (data release 1 or 2), the Gaia archive table to be used in the search and the name of the output file. When the shape of the query is a cone, the user must enter the value of the radius in degrees. If the option chosen is a box, the user must enter the height and width in degrees.



Finally, if the choice is a polygon, the user must fill in the value of the vertices separated by commas,



  • Equatorial Coordinates

The default option is by equatorial coordinates. In this case the user must fill in the parameters of the coordinates Right ascension (RA) and Declination (DEC). The rest of the options are common to the Name case.



  • File

Another interesting option is to make the query through a file. To do this, the user must choose the “File” option and upload the file using the “Browse…” button (depending on the browser this button can have a different caption).



  • ADQL

The last available option is through the use of the ADQL language. By default, choosing this option displays an example of a query.



3.2 - Herschel

The method to make queries to this archive is similar to that described in the previous section. The main difference lies in the choice of device (PACS, SPIRE or HIFI) in the case of queries by Name and Equatorial coordinates.



3.3 - Queries Status

Once any of the queries are launched by clicking on the “Submit” button, the application will automatically load the “Queries Status” page,



The operation and options are similar to those explained in 2.2 Pineline Status with some differences:

  • The status column has four possible status: Pending, Started, Error and Success.
  • In this case, the “Launch” button opens a further drop-down field to choose an available algorithm that will run over the results of the selected query Name,


Algorithm selection to run over the selected results of a query.

When an algorithm is selected, the configuration panel of the chosen algorithm is shown. It is also possible to configure the algorithm parameters from here and launch the algorithm,



It is possible to launch algorithms directly from the “Queries Status” page.

4 - Data

4.1 - Content

Within the “Data” section of the side menu, the user can navigate and interact with the downloaded or generated files. For this, the user must click on “Data” and then on “File Manager”.



Every user has three folders:

  • Sources: This folder in turn contains the files obtained from Gaia and Herschel (hsa) queries.
  • Results: It contains all the files generated by the algorithms as results.
  • User: It contains any files uploaded by the user.

4.3 - Upload and download and delete files
  • Upload files

DEAVI allows the user to upload files in two different ways:

  1. Through the web interface of the application: The user has to select the file by clicking on the button


Then, the user must click on the “upload file” button,



and the file will be uploaded.

  1. Through the SAMP protocol. In this case, the user must first register the hub by clicking on the “Register” button,


Then, the status will change to “Yes” and the “Unregister” button will be activated if the hub is active. Otherwise, the application will inform the user that it has not located the hub.



Note that the user must deactivate the mixed content of the web browser used.

  • Delete files

When the user views a certain file, its name, its size and the possibility of showing the content, downloading or deleting it are displayed.



  • Show content: When the user clicks on the file name, a new browser window is opened showing its contents.
  • Delete File: If the user clicks on the “Delete” button,


a pop up will appear where it is reported that the file will be deleted if the user clicks on “Yes”. If the user clicks on “No”, the pop up will be closed without deleting the file.



  • Download files

Finally, if the user clicks on the “Download” button,



the file will be downloaded to a local folder.

5 - Help

The last section of DEAVI is the “Help” section. In this section the user can find information on how to use DEAVI, “Help” as well as the information of the consortium members of the SFM project, “About”.
